## 左侧 · 助手（Assistant）

助手 = 一个独立的「人设 + 默认模型 + 系统提示词 + 工具配置」组合。可以理解为「自定义 GPT」或「Claude Project」。

### 默认助手

ProLab 初始自带一个「**默认助手**」（带 ⭐ 标记表示星标 / 默认），无人设、使用通用配置，适合日常问答。

### 新建助手

点击底部「**+ 新建助手**」打开配置面板（图未列出，按以下字段填写）：

| 字段 | 示例 | 用途 |
| --- | --- | --- |
| 名称 | `代码审查员` | 显示在助手列表 |
| 头像 | 上传图片或选 Emoji | 视觉区分 |
| 默认模型 | `claude-sonnet-4-6` | 该助手下新建对话自动使用 |
| 系统提示词 | `你是资深 Go 工程师...` | 影响所有该助手下的对话风格 |
| 思考模式默认 | 开 / 关 | 进入对话时是否自动开启 |
| 联网模式默认 | 开 / 关 | 同上 |

<Callout type="info" title="助手的应用场景">
- 「代码审查员」专家：模型 Claude + 系统提示词「严格审查代码安全 / 性能」；
- 「翻译官」：模型 GPT + 系统提示词「中英互译，保持原文语气」；
- 「日常助手」：默认助手即可，通用问答。
</Callout>

## 中部 · 对话区

### 输入栏字段

{/* 图片占位 · 可与 Image #9 共用，或单独裁剪底部输入栏区域。建议路径：/assets/image/ProLab/07-chat-input.webp */}

底部输入栏从左到右：

| 控件 | 功能 |
| --- | --- |
| **选择模型** 下拉 | 临时切换当前对话使用的模型（覆盖助手的默认模型） |
| 💡 **思考** 开关 | 启用 Thinking / Reasoning 模式（支持 Claude / Codex / DeepSeek-R1 等） |
| 🌐 **联网** 开关 | 启用联网检索，模型可访问实时网页 |
| 📎 **附件** | 上传图片 / PDF / 文档（取决于模型多模态能力） |
| **输入框** | Enter 发送，Shift+Enter 换行 |

<Callout type="warn" title="思考 / 联网需要模型支持">
- **思考**：仅 `claude-*-thinking`、`o1-*`、`deepseek-reasoner`、`gemini-*-thinking` 等支持；
- **联网**：仅特定模型或上游通道开启（如 ProAPI 部分模型预置联网工具）。

切换到不支持的模型时，开关会自动置灰。
</Callout>

### 消息流

- **用户消息** 显示在右侧，**助手消息** 显示在左侧；
- 助手消息上有操作按钮：复制、重新生成、分支（fork）、删除；
- 代码块自带高亮 + 一键复制；
- Markdown / LaTeX / Mermaid 图自动渲染。

## 右侧 · Sessions（对话历史）

每个助手下有独立的对话历史，分两类管理：

<Tabs items={["永久 Sessions","临时 Sessions"]}>

<Tab value="永久 Sessions">

**特点：**
- 永久保存在浏览器（IndexedDB）
- 显示创建日期
- 可手动重命名 / 删除
- 适合**需要后续翻看 / 续聊**的对话

**操作：**
- 点击列表项切换到该对话；
- 右键 / 悬停可重命名、删除、导出 Markdown。

</Tab>

<Tab value="临时 Sessions">

**特点：**
- **关闭浏览器自动清除**
- 不出现在永久列表中
- 适合**一次性问答 / 测试 / 不想留痕**的场景

**使用：**
- 在 Sessions 顶部点「临时」Tab；
- 「+ 新建对话」时勾选「临时」即可。

</Tab>

</Tabs>

<Callout type="info" title="侧栏折叠">
左右两栏顶部都有折叠按钮（▢），适合在写长 Prompt 时把两栏都收起，获得更大的对话视野。
</Callout>

## 推荐使用方式

### 场景 1：日常问答

直接用「默认助手 → 临时 Session」，关掉思考、关掉联网，模型选 `gpt-4o-mini` 或 `claude-haiku-4-5` 这类便宜快速的，**省钱省时**。

### 场景 2：代码 / 技术深度问题

1. 新建「**代码助手**」，模型 `claude-sonnet-4-6`，系统提示词写明技术栈；
2. 开启 **思考模式**；
3. 切到「**永久 Session**」保留对话脉络，方便后续回顾。

### 场景 3：调研 / 时效性问题

1. 模型选支持联网的（如 `gpt-4o-search` / `claude-*-with-search`）；
2. **开启联网模式**；
3. 多轮追问，让模型逐步钻取细节。

### 场景 4：批量整理 / 翻译

1. 新建「**翻译官**」助手，系统提示词锁定语言对；
2. 临时 Session，连续粘贴多段文本；
3. 用完关掉浏览器自动清理。

## 常见问题

<Accordions>

<Accordion title="Q1：思考模式和普通模式有什么区别？">

思考模式（Reasoning / Thinking）让模型**先输出内部推理过程再给最终答案**。优点：复杂数理 / 代码 / 推理题正确率显著提升；缺点：输出更长、token 消耗 1.5 ~ 3 倍。简单问答关闭即可。

</Accordion>

<Accordion title="Q2：联网检索调用的是哪个搜索引擎？">

取决于模型与上游。ProAPI 转发到对应模型自带的检索能力（OpenAI / Claude / Gemini 各家工具不同），ProLab 本身不接入第三方搜索。

</Accordion>

<Accordion title="Q3：附件上传支持哪些格式？">

由所选模型决定：
- **图片**：`png` / `jpg` / `webp`（多模态模型如 GPT-4o / Claude / Gemini 都支持）；
- **PDF / DOCX / TXT**：部分模型原生支持，其余由 ProLab 前端转文本注入；
- **代码 / 数据文件**：纯文本嵌入对话上下文。

</Accordion>

<Accordion title="Q4：能在助手之间复用同一份 Session 吗？">

不能。Sessions 严格归属于某个助手（因为系统提示词和默认模型绑定在助手上）。如需「换助手继续聊」，可以**复制对话内容**到新助手的新 Session。

</Accordion>

<Accordion title="Q5：永久 Sessions 占满浏览器存储怎么办？">

IndexedDB 通常有 GB 级配额，文本对话很难占满。但如果有大量附件，可：
- 「设置 → 通用设置」中查看存储用量；
- 定期清理不再需要的旧 Sessions；
- 重要对话「导出 Markdown」后再删除。

</Accordion>

<Accordion title="Q6：能多端同步对话历史吗？">

可以。「设置 → 提示词同步」开启账号同步后，对话历史与提示词库会同步到云端（端到端加密），多端登录自动拉取。

</Accordion>

</Accordions>
