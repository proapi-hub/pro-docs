## 先确认上游是不是启用了

![ProAPI 上游已启用](/assets/image/ProLab/04-upstream-active.webp)

调用失败时，先看 **设置 → 上游配置**。  
这张图里 ProAPI 卡片已经出现，并且能看到模型数量，说明令牌和模型导入至少已经完成。

如果你这里看不到 ProAPI 卡片，或者模型数量明显不对，先不要继续测对话或创作，回到 [5 分钟接入 ProAPI](/prolab/quick-start) 重新接入。

## 再用模型测试定位问题

![模型测试页](/assets/image/ProLab/04-model-test.webp)

这张图是 **设置 → 模型测试**。新手主要看 4 个区域：

- **模型分类**：先选对话、绘图或视频，不要混在一起测；
- **筛选模型**：输入模型名，只测目标模型；
- **测试提示词**：保持默认也可以，先确认能返回；
- **结果列表**：成功、失败、耗时都会显示在这里。

<Callout type="info" title="先测一个模型">
不要第一次就批量测试所有模型。先筛选一个模型，确认令牌和网络没问题，再扩大测试范围。
</Callout>

## 错误应该怎么判断

![模型测试页](/assets/image/ProLab/04-model-test.webp)

模型测试失败时，先看错误类型：

| 错误现象 | 通常说明 | 下一步 |
| --- | --- | --- |
| `401` | 令牌错误、过期或复制不完整 | 重新复制 ProAPI 令牌 |
| `403` | 当前令牌分组不包含该模型 | 换正确分组或全模型令牌 |
| 超时 | 网络、代理或上游响应慢 | 降低并发，检查代理 |
| 模型不存在 | 没有导入该模型，或模型名不匹配 | 回上游配置重新导入 |

## 其他设置入口怎么看

![模型测试页](/assets/image/ProLab/04-model-test.webp)

截图左侧是设置页菜单。除了 **上游配置** 和 **模型测试**，常见入口还有：

| 入口 | 什么时候点 |
| --- | --- |
| **代理配置** | 访问 ProAPI 不稳定或需要固定网络出口 |
| **导入 / 导出** | 换电脑、备份配置、迁移助手和对话 |
| **Prompt 设置** | 调整标题生成、压缩、建议问题等系统提示词 |
| **提示词同步** | 更新或同步提示词库 |
| **MCP 服务** | 给助手增加工具调用能力 |
| **通用设置** | 设置默认模型、防窥、标题长度等偏好 |

<Callout type="warn" title="导出配置前先检查密钥">
导入 / 导出的配置可能包含上游信息。分享给别人之前，确认里面没有真实 API Key。
</Callout>
