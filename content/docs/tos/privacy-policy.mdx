## 平台核心隐私承诺

<Callout type="info" title="核心承诺">
  作为中转聚合平台，我们坚决 **不持久化存储** 您发送给模型的提示词（Prompt）以及模型返回的生成内容（无论文本、图像还是视频）。所有业务数据仅在服务器内存中完成即时转发。我们仅记录实现 **计费和基础安全防御** 所必需的元数据（如耗费额度、模型名称、调用时间戳）。
</Callout>

> 您的信任是我们存在的基石。

## 1. 引言与核心原则

欢迎使用本平台提供的应用程序编程接口（API）聚合服务。我们深知数据隐私对开发者的重要性，因此在系统架构设计之初，就将 **「数据最小化」** 和 **「无状态中转」** 作为最高指导原则。

本《隐私政策》旨在向您透明地解释我们在您使用服务期间，如何收集、使用、保护以及处理您的个人信息和业务数据。

## 2. 我们收集哪些信息

为了向您提供稳定、合规的服务，我们 **仅收集绝对必要** 的信息：

<Accordions>
<Accordion title="账户凭据">

当您注册时，我们需要收集您的 **电子邮箱地址** 或 **第三方授权登录标识**（如 GitHub 唯一 ID），用于创建账户和身份验证。

</Accordion>
<Accordion title="账单与支付信息">

当您进行预付费充值时，我们的 **第三方支付网关** 会处理您的交易。我们仅接收并记录交易的 **流水号和金额状态**，**绝不收集或存储您的信用卡号或敏感金融信息**。

</Accordion>
<Accordion title="系统调用日志">

当您调用我们的接口时，为实现精确计费和防滥用拦截，系统会自动生成 **不包含业务正文** 的元数据日志，包含：

- 您的 IP 地址
- 请求时间戳
- 请求消耗的计算额度（Token 数量）
- 请求的模型种类
- 接口响应状态码

</Accordion>
</Accordions>

## 3. 信息的使用目的

我们收集上述基本信息的目的 **仅限于** 保障服务的正常运转：

1. 提供、维护和改进本平台的聚合接口服务。
2. 处理您的支付流水，计算账户额度消耗。
3. 识别、调查和预防欺诈行为、恶意刷单请求及任何违反我们《用户协议》的技术滥用。
4. 通过您的邮箱向您发送重要的服务变更通知（如模型下线、系统维护公告、余额不足提醒）。

## 4. 模型数据流转说明（核心）

理解我们在业务链条中的角色对您的隐私保护至关重要。

### 4.1 无状态的内存转发

当您发起生成请求时，您的完整请求体（包含上下文、系统指令、提示词）会到达我们的边缘网关，随后被 **即时且加密地转发** 给后端的实际模型供应商（如 OpenAI、Google、Anthropic）。供应商返回的生成结果也会通过我们 **即时回传** 给您。

<Callout type="info" title="技术保障">
  整个过程中，您的输入与输出数据 **仅存在于我们服务器的瞬时内存中**，一旦网络请求生命周期结束，数据即被从内存中销毁。

  **我们没有甚至不具备查看或审查您具体业务数据的数据库能力**。
</Callout>

### 4.2 上游供应商的隐私政策

请注意，您的业务最终是由底层的第三方大语言或多模态模型进行计算的。因此，这些业务数据的最终隐私处理规范，**受限于对应上游厂商的隐私政策约束**。

<Callout type="warn" title="阅读上游政策">
  我们 **强烈建议** 您在调用特定模型前，了解该模型原厂商的数据保留政策（例如，许多厂商目前承诺不使用 API 数据来训练其下一代模型，但具体条款随时可能变更）。
</Callout>

### 4.3 数据出境合规义务

若您从中国大陆或其它对数据出境有强制要求的司法管辖区使用本服务，**通过本服务发送至境外模型的请求构成数据出境行为**。您应自行履行《个人信息保护法》《数据安全法》等规定的：

- 数据出境安全评估
- 个人信息主体的单独同意
- 跨境数据传输备案

平台作为海外技术中转方，**不承担用户数据出境合规义务**。

## 5. 信息共享与披露

我们承诺：**绝不出售、出租或交易您的任何个人信息与调用习惯**。我们仅在以下极其有限的情况下披露信息：

### 5.1 必要的基础设施供应商

我们可能会向提供核心技术支持（如云服务器托管、邮件发送服务、支付处理）的受信任第三方共享 **极少量的脱敏数据**，且他们受到严格的保密协议约束。

### 5.2 法律强制与合规调查

响应司法管辖区内具有法律强制力的：

- 法院传票
- 警方搜查令
- 监管部门的正式调查要求

在法律允许的范围内，我们将尝试在披露前通知您。

## 6. 数据存储与安全保护

我们采取了符合行业标准的技术和组织措施来保护您的信息安全：

| 保护层 | 措施 |
| --- | --- |
| 传输加密 | 客户端 ↔ 平台、平台 ↔ 上游模型之间的所有通信均强制使用最先进的 **TLS（传输层安全性）加密协议** |
| 密钥单向加密 | 您的登录密码采用强哈希算法（如 **bcrypt**）进行单向不可逆加密存储 |
| 接口凭证隔离 | 您生成的调用接口密钥（`sk-key`）直接与账户余额绑定 |

<Callout type="warn" title="密钥保管责任">
  请您务必妥善保管接口密钥。若发现异常调用，请立即在控制台 **作废该密钥并重新生成**。密钥泄露后产生的余额损失由用户自行承担。
</Callout>

## 7. 您的用户权利

您对您的账户和个人信息拥有完全的控制权：

<Accordions>
<Accordion title="查看与更正">

您有权随时登录控制台，**查看、更正或更新** 您的注册电子邮箱及密码。

</Accordion>
<Accordion title="数据导出">

您可以随时导出：

- 您的充值账单
- 近 **30 天** 的调用日志记录

</Accordion>
<Accordion title="账户注销">

您有权请求 **彻底删除** 您的账户。账户注销后：

- 除法律法规要求必须留存的交易记录外，您的所有注册信息将被 **永久抹除**；
- 注销操作 **不可恢复**；
- 账户余额按《用户协议》第 4.2 节处理（原则上不退款）。

</Accordion>
</Accordions>

## 8. 政策变更与联系方式

### 8.1 政策变更

随着业务的发展及法律法规的更新，我们可能会不定期修订本隐私政策。对于 **重大变更**，我们将在控制台显著位置发布公告或通过邮件通知您。

### 8.2 联系我们

如果您对本隐私政策或我们的数据处理实践有任何疑问、投诉或建议，请随时通过控制台内的官方客服入口联系我们。我们将尽力在最短时间内为您解答。
