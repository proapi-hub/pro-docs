## 概览

`seedance2.0-720p` 与 `seedance2.0-fast-720p` 接入字节官方 Seedance 2.0 上游(720p 输出端点),相比本站 `seedance2.0-5s/10s` 等锁档版本能力更细:

- 时长可变 `5`–`15` 秒
- 最多 `4` 张参考图(`reference_images`)
- 首尾帧(`reference_mode=start_end`)

Base URL:`https://newapi.prorisehub.com`,鉴权头:

```http
Authorization: Bearer sk-<音视频分组令牌>
Content-Type: application/json
```

<Callout type="warn" title="先看这两条">
  - **模型名必须带 `-720p` 后缀**。本站目前对接的是字节 Seedance 2.0 的 **720p 输出端点**,模型名 `seedance2.0-720p` / `seedance2.0-fast-720p` 是不可省略的全名。漏写 `-720p` 会返回 `model not found`。
  - **当前仅支持 4 图参考 + 首尾帧**。视频参考(`reference_video` / `reference_videos`)、音频参考(`audio_url`)等高级能力**当前不支持**,传了会被上游忽略或报错。
</Callout>

<Callout type="info" title="后续会上新更多档位">
  我们正在对接支持 **1080p / 480p 多档位、视频参考、音频参考** 的 Seedance 上游,届时会以新模型名(如 `seedance2.0-1080p`、`seedance2.0-fast` 等)上架,不会动这两条已有模型的契约。文档会同步更新。
</Callout>

<Callout type="warn" title="必须用音视频分组令牌">
  本页所有调用必须使用 **音视频模型** 分组令牌。其它分组令牌返回 `model not found`。
</Callout>

## 模型与计费

| 模型 | ¥/秒 | 时长(`seconds` / `duration`) | 整条价 ¥ | 定位 |
| --- | ---: | --- | --- | --- |
| `seedance2.0-fast-720p` | `0.15` | `5` – `15` | `0.75`–`2.25` | 快速档,迭代成本低 |
| `seedance2.0-720p` | `0.30` | `5` – `15` | `1.50`–`4.50` | 标准档,画质更稳 |

显示口径:1 USD = 1 ¥;按秒计费,`¥/秒 × 视频时长`。

## 能力矩阵

| 维度 | 支持范围 |
| --- | --- |
| 分辨率 | **`720p` 固定**(上游 720p 端点,暂不支持 480p / 1080p) |
| 画幅 `aspect_ratio` | `16:9` `9:16` `1:1` `4:3` `3:4` `21:9` |
| 时长 `duration` | `5`–`15` 秒整数 |
| 参考图 `reference_images` | 最多 `4` 张(公网 HTTPS URL) |
| 首尾帧 `reference_mode=start_end` | 必须传 `2` 张参考图,按 `[首帧, 尾帧]` 顺序 |
| 视频自带音频 | 默认开启 |
| 视频参考 `reference_video(s)` | ❌ **暂不支持**(后续模型会开放) |
| 音频参考 `audio_url` | ❌ **暂不支持**(后续模型会开放) |
| 1080p / 480p | ❌ **暂不支持**(后续模型会开放) |

## `reference_mode` 模式

| 模式 | 说明 |
| --- | --- |
| `auto` | 默认;系统按传入的参考图数量自动选择处理方式 |
| `start_frame` | 首帧模式,**必须且只能**传 1 张 `reference_images` |
| `start_end` | 首尾帧模式,**必须且只能**传 2 张 `reference_images` |
| `image_reference` | 多图参考模式,可传 1–4 张 `reference_images` |

## 图生视频(最多 4 图参考)

```bash
curl https://newapi.prorisehub.com/v1/videos \
  -H "Authorization: Bearer sk-音视频分组令牌" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance2.0-720p",
    "prompt": "Turn these stills into a polished fashion film with subtle camera motion",
    "duration": 8,
    "reference_images": [
      "https://example.com/look-1.jpg",
      "https://example.com/look-2.jpg",
      "https://example.com/look-3.jpg",
      "https://example.com/look-4.jpg"
    ],
    "aspect_ratio": "16:9",
    "reference_mode": "image_reference"
  }'
```

`reference_images` 仅接受公网 URL(HTTPS),最多 4 张。请勿使用本地文件上传或 `data:` URI,避免请求体过大。

## 首尾帧

```bash
curl https://newapi.prorisehub.com/v1/videos \
  -H "Authorization: Bearer sk-音视频分组令牌" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance2.0-720p",
    "prompt": "Smooth transition from day to night",
    "duration": 6,
    "reference_images": [
      "https://example.com/day.jpg",
      "https://example.com/night.jpg"
    ],
    "aspect_ratio": "16:9",
    "reference_mode": "start_end"
  }'
```

<Callout type="warn">
  `start_end` 模式必须且只能传 2 张 `reference_images`,顺序为 `[首帧, 尾帧]`。不能同时传 `reference_video` / `reference_videos`(当前模型也不支持视频参考),否则返回 `400 invalid_reference_inputs`。
</Callout>

## 查询任务状态

```bash
curl https://newapi.prorisehub.com/v1/videos/<task_id> \
  -H "Authorization: Bearer sk-音视频分组令牌"
```

| `status` | 含义 |
| --- | --- |
| `queued` | 已入队 |
| `processing` | 生成中(`progress` 0–100) |
| `succeeded` / `completed` | 完成,可用 `url` 或 `/content` 下载 |
| `failed` | 失败,无扣费 |

完成响应示例:

```json
{
  "id": "task-abc123",
  "object": "video",
  "model": "seedance2.0-fast-720p",
  "status": "succeeded",
  "progress": 100,
  "seconds": "5",
  "size": "1280x720",
  "url": "https://newapi.prorisehub.com/v1/videos/task-abc123/content",
  "created_at": 1713168000,
  "completed_at": 1713168120
}
```

## 下载视频

```bash
curl -L https://newapi.prorisehub.com/v1/videos/<task_id>/content \
  -H "Authorization: Bearer sk-音视频分组令牌" \
  -o result.mp4
```

任务未完成时返回 `409`,错误码 `video_not_ready`。

## 兼容字段

| 字段 | 行为 |
| --- | --- |
| `duration` | **推荐**。整数 5–15 |
| `seconds` | 兼容字段;仅在未传 `duration` 时生效 |
| `image_url` / `reference_image_urls` | 合并到 `reference_images`(最多累计 4 张) |
| `referenceImages` | `reference_images` 驼峰兼容字段 |

同时传 `duration` 与 `seconds` 时以 `duration` 为准。

<Callout type="info" title="视频/音频参考字段">
  `reference_video` / `reference_videos` / `audio_url` 及对应驼峰别名当前模型**不会处理**;传了不会报错但也不会生效。等支持这些能力的新模型上架后再用。
</Callout>

## 错误码

| HTTP | 错误码 | 说明 |
| --- | --- | --- |
| `400` | `invalid_json` | JSON 格式错误 |
| `400` | `model_not_supported` | 模型不存在或非视频模型(常见原因:模型名漏了 `-720p` 后缀) |
| `400` | `invalid_reference_inputs` | 参考图数量与 `reference_mode` 不匹配 |
| `400` | `invalid_reference_mode` | `reference_mode` 不支持 |
| `400` | `invalid_resolution_name` | 分辨率不支持(当前固定 720p,传 1080p / 480p 会报这个) |
| `400` | `invalid_video_length` | 时长不在 5–15s 整数范围 |
| `404` | `task_not_found` | 任务不存在 |
| `409` | `video_not_ready` | 视频尚未完成 |
| `429` | `upstream_rate_limited` | 请求过于频繁 |
| `451` | `upstream_safety_rejected` | 内容或素材触发安全策略 |
| `503` | `token_unavailable` | 暂无可用上游 |
| `504` | `upstream_timeout` | 生成超时 |

错误响应:

```json
{
  "error": {
    "message": "错误描述",
    "type": "invalid_request_error",
    "param": "参数名",
    "code": "错误码"
  }
}
```
