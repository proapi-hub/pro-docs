## 基本信息

| 项 | 值 |
| --- | --- |
| **API 域名** | `https://newapi.prorisehub.com` |
| **认证方式** | `Authorization: Bearer <你的令牌>` |
| **请求格式** | JSON（`Content-Type: application/json`），仅图片编辑接口走 `multipart/form-data` |
| **响应格式** | JSON；流式接口走 SSE (`text/event-stream`)；视频内容下载返回 `video/mp4` |
| **协议** | HTTPS only |

<Callout type="info" title="先创建令牌">
  所有 API 调用都需要令牌。请先参照 [令牌与分组](/token/intro) 创建合适分组的令牌再回到本章节。**令牌的分组决定了你能调用哪些模型**。
</Callout>

## 当前公开能力

当前站点公开价格接口显示 4 类主要入口：

| 能力 | 方法与路径 | 适合谁 |
| --- | --- | --- |
| OpenAI 兼容聊天 | `POST /v1/chat/completions` | OpenAI SDK、Cherry Studio、Cline、OpenCode、Cursor 等 |
| Anthropic 原生消息 | `POST /v1/messages` | Claude Code、Claude Desktop、Cline 的 Anthropic 配置 |
| Gemini 原生生成 | `POST /v1beta/models/{model}:generateContent` | Gemini CLI、Google Gemini SDK |
| 图片生成 | `POST /v1/images/generations` | OpenAI Images 兼容客户端 |

除此之外，站点也提供视频异步任务接口：

| 能力 | 方法与路径 | 说明 |
| --- | --- | --- |
| 创建视频任务 | `POST /v1/videos` | 返回任务 ID，适合视频生成 |
| 查询视频任务 | `GET /v1/videos/{video_id}` | 查看任务状态、进度和结果 |
| 下载视频内容 | `GET /v1/videos/{video_id}/content` | 完成后返回 `video/mp4`，支持浏览器和客户端直接下载 |

## 查询当前模型和价格

如果你要让程序自动判断“有哪些模型”“属于哪个分组”“大概怎么计费”，可以读公开价格接口：

```bash
curl https://newapi.prorisehub.com/api/pricing
```

这个接口不需要 API Key，返回内容里最常用的是：

| 字段 | 说明 |
| --- | --- |
| `data` | 模型列表，包含模型名、说明、分组、价格和支持的端点类型 |
| `group_ratio` | 当前分组倍率 |
| `usable_group` | 当前可选分组及说明 |
| `supported_endpoint` | 当前公开支持的主要端点 |

<Callout type="info" title="给自动化脚本的建议">
不要把文档里的模型列表写死。更稳的做法是：启动时读取 `/api/pricing`，再结合用户令牌所在分组筛选可用模型。
</Callout>

## 通用请求示例

```bash
curl https://newapi.prorisehub.com/v1/chat/completions \
  -H "Authorization: Bearer sk-你的令牌" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": false
  }'
```

## 错误码

| HTTP 状态码 | 含义 | 排查建议 |
| --- | --- | --- |
| `401 Unauthorized` | 令牌错误 / 已禁用 | 检查 `Authorization` 头格式是否为 `Bearer sk-xxx`，并确认令牌未被删除或禁用 |
| `403 Forbidden` | 模型不在令牌分组内 | 在控制台「模型广场」确认当前分组是否包含目标模型，或更换分组令牌 |
| `400 model_price_error` | 模型未配置价格 | 联系管理员补充定价；通常出现在新接入但未上架的模型 |
| `429 Too Many Requests` | 触发分组限流 | 等待 1 分钟或单分钟内降低并发 |
| `500 / 502 / 503` | 上游错误 | 偶发可重试；持续出现请到 [check.linux.do/group/Pro](https://check.linux.do/group/Pro) 查看监控 |
| `504 Gateway Timeout` | 网关超时（Cloudflare 100s 限制） | 图片 / 视频生成耗时长，请参考 [音视频模型章节](/api/media) 的长连接说明 |

错误响应示例：

```json
{
  "error": {
    "message": "specific error description",
    "type": "new_api_error",
    "code": "model_price_error",
    "param": "",
    "request_id": "..."
  }
}
```

<Callout type="info" title="排障关键字段">
  `request_id` 是排障的关键。遇到无法解决的错误时，请将它与请求时间一起反馈给管理员。
</Callout>

## 流式响应（Stream）

聊天 / 对话类接口支持 `stream: true`，返回 Server-Sent Events 流：

```
data: {"id":"chatcmpl-...","object":"chat.completion.chunk","choices":[{"delta":{"content":"你"}}]}
data: {"id":"chatcmpl-...","object":"chat.completion.chunk","choices":[{"delta":{"content":"好"}}]}
data: [DONE]
```

<Callout type="warn" title="图像 / 视频不支持流式">
  请不要在 `/v1/images/*` 和音视频模型上设置 `stream: true`，会被上游拒绝。
</Callout>

## 速率与并发

- ProAPI 不设统一的全局限流，但**每个分组各自的上游会有限流**（如 Anthropic / Google 官方）。
- 单条请求最长执行时间受 Cloudflare 网关 **100 秒** 限制，触发后会返回 `504`。
- 视频生成本身耗时 3 ~ 6 分钟，**必须** 使用直连源站 + 本地客户端长连接策略，详见 [音视频模型章节](/api/media)。
