ProAPI 音视频分组对外暴露统一、稳定的模型名（如 `nano-banana`、`sora-2`），由 ProAPI 网关根据请求中的 `quality` / `aspect_ratio` / `duration` 等参数，自动选择最匹配的后端实现。客户端**只需关心模型名 + 业务参数**，无需关心上游具体型号、版本号或分辨率档位。

<Callout type="info" title="抽象层稳定承诺">
本章列出的模型名与参数语义即 ProAPI 对外契约。上游号池、分辨率档位等实现细节由网关接管，可能随时调整、聚合或下线，**不影响你的客户端代码**。计费严格按你请求中的 `model` 字段固定，不会因后端切换型号而变价。
</Callout>

## 端点

| 用途 | 推荐端点 | 备选 |
| --- | --- | --- |
| 图像（文生图） | `POST /v1/images/generations` | `POST /v1/chat/completions` |
| 图像（图生图 / 编辑） | `POST /v1/images/edits` | – |
| 视频（文 / 图生视频） | `POST /v1/chat/completions` | – |

所有请求使用 **音视频模型** 分组令牌：

```
Authorization: Bearer sk-<你的音视频分组令牌>
Content-Type: application/json
```

域名固定为 `https://newapi.prorisehub.com`。详细鉴权规则见 [OpenAI 兼容端点](/api/openai)。

## 请求参数

ProAPI 在 OpenAI 兼容协议之上扩展了少量与媒体生成强相关的字段。下面以参数为单位说明语义；具体每个模型的取值范围参考下方 [取值范围矩阵](#取值范围矩阵)。

### 通用字段（OpenAI 协议）

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `model` | string | ✅ | 上方清单中的模型名 |
| `prompt` / `messages` | string / array | ✅ | 文生图走 `prompt`，视频走 `messages`，长度至少 3 个字符 |
| `n` | integer | ❌ | 仅图像，**仅支持 `1`** |
| `size` | string | ❌ | 仅图像，OpenAI 标准尺寸字段，可由网关派生 `aspect_ratio`，详见下文 |
| `stream` | bool | ❌ | 视频生成始终非流式，**请显式传 `false`** |

### ProAPI 扩展字段

| 参数 | 类型 | 适用 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `quality` | string | 图像全部 / `veo-3.1*` | `medium` | 控制输出清晰度档位 |
| `aspect_ratio` | string | 全部 | `16:9` | 控制画面比例 |
| `duration` | integer | 视频全部 | 见上表 | 控制视频时长（秒） |

`quality` 取值（语义档位）：

| 档位 | 同义词 | 说明 |
| --- | --- | --- |
| `low` | `standard` | 基础档，最快 |
| `medium` | `hd` | 通用档，**默认** |
| `high` | `ultra` | 高清档 |

<Callout type="info" title="为什么 ProAPI 用语义档位而不是分辨率数字">
不同模型的"高清"对应的实际分辨率不同（图像 4K vs 视频 1080p 等）。统一用 `low` / `medium` / `high` 三个语义档位，客户端无需理解每个模型的具体规格，**切模型时无需重写参数**。
</Callout>

### `size` 字段（OpenAI 协议 — 仅图像）

完全遵循 OpenAI Images API，**只识别协议中真实存在的 7 个尺寸**：

| `size` | 派生 `aspect_ratio` | 协议来源 |
| --- | --- | --- |
| `1024x1024` | `1:1` | gpt-image / dall-e |
| `1024x1536` | `2:3` | gpt-image |
| `1536x1024` | `3:2` | gpt-image |
| `1024x1792` | `9:16` | dall-e-3 |
| `1792x1024` | `16:9` | dall-e-3 |
| `256x256` | `1:1` | dall-e-2 |
| `512x512` | `1:1` | dall-e-2 |

<Callout type="warn" title="非协议 size 不会被识别">
传 `3840x2160`、`2048x2048` 等 OpenAI 协议外的尺寸将走默认值。要控制 4K 或自定义比例，请改用 `quality` + `aspect_ratio` 扩展字段。
</Callout>

`size` 与 `aspect_ratio` 同时传时，**`size` 派生值优先**。建议只用其中一种。

### 取值范围矩阵

#### 图像模型

| 模型 | 支持的 `aspect_ratio` |
| --- | --- |
| `nano-banana` / `nano-banana-pro` | `1:1` `16:9` `9:16` `4:3` `3:4` |
| `nano-banana-2` | 上述 + `1:4` `4:1` `1:8` `8:1` |
| `gpt-image-2` | `1:1` `5:4` `4:5` `9:16` `21:9` `16:9` `3:2` `2:3` `4:3` `3:4` |
| OpenArt 系列<sup>†</sup> | `1:1` `16:9` `9:16` `4:3` `3:4` `3:2` `2:3` |

<small>† OpenArt 系列指 `flux-2-*`、`google-imagen-4`、`byte-plus-seedream-*`、`qwen-image-*`、`wan2-7-image*`、`grok-imagine`，详细比例支持以模型广场实时检测为准。</small>

所有图像模型均支持 `quality` 三档。

#### 视频模型

| 模型 | `duration` 取值 | `aspect_ratio` | `quality` |
| --- | --- | --- | --- |
| `sora-2` | `4` / `8` / `12` | `16:9` `9:16` | – |
| `sora-2-pro` | `4` / `8` / `12` | `16:9` `9:16` | – |
| `kling-3.0` | `5` / `10` / `15` | `16:9` `9:16` | – |
| `kling-o3` | `5` / `15` | `16:9` `9:16` | – |
| `veo-3.1` | `4` / `6` / `8` | `16:9` `9:16` | 支持三档（影响输出分辨率） |
| `veo-3.1-fast` | `4` / `6` / `8` | `16:9` `9:16` | 支持三档 |
| `veo-3.1-ref` | `4` / `6` / `8` | `16:9` `9:16` | 支持三档 |

未列出的字段对该模型无效，传了会被忽略。

<Tab value="Python · 图像">

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-<音视频分组令牌>",
    base_url="https://newapi.prorisehub.com/v1",
    timeout=240,
)

resp = client.images.generate(
    model="nano-banana-pro",
    prompt="一只柴犬在富士山下奔跑，浮世绘风格",
    n=1,
    extra_body={
        "quality": "high",
        "aspect_ratio": "16:9",
    },
)
print(resp.data[0].url)
```

</Callout>

<Callout type="info" title="固定价计费">
音视频模型采用 **固定价计费**：单次调用价格与 `quality` / `duration` / `aspect_ratio` 无关，仅与 `model` 字段相关。最终扣费 = 模型单价 × 分组倍率。
</Callout>

## 超时与长连接

视频生成是同步桥接，**实际耗时通常 150 ~ 400 秒**。

<Callout type="warn" title="Cloudflare 100 秒硬限制">
公网域名 `newapi.prorisehub.com` 走 Cloudflare 代理，**所有请求最长 100 秒**，超时返回 `504`。这是 Cloudflare 免费 / Pro 计划的硬限制。

视频调用请处理：

1. 客户端 `timeout` 设置 ≥ 400 秒（参考 Cherry Studio、CC-Switch、Claude Code 各自的超时配置）；
2. 将 ProAPI 域名加入本地代理直连白名单，避免代理切断长连接；
3. 100 秒固定返回 `504` 几乎都是 Cloudflare 网关问题，与 ProAPI 服务可用性无关。
</Callout>

## 错误码速查

| 状态码 | `error_code` | 含义 / 处理 |
| --- | --- | --- |
| `400` | `validation_error` | 参数不合法。`prompt` 过短（< 3）、`aspect_ratio` 不在该模型支持列表、`duration` 越界等 |
| `400` | `bad_request` | 模型要求参考图但未传（典型：`kling-3.0` / `veo-3.1-ref`） |
| `404` | `model_not_found` | 模型名拼写错误或不在 `/v1/models` 列表中 |
| `429` | `rate_limit_exceeded` | 触发模型 RPM / 用户配额限制，参考响应头 `Retry-After` 退避 |
| `504` | – | Cloudflare 网关超时（见上节） |

## 常见问题

<Accordions>

<Accordion title="Q1：响应里的 model 字段值是什么？">

`response.model` 为 ProAPI 内部规范化后的标识，可能与请求体中的 `model` 不完全一致。**计费始终按请求中的 `model` 执行**，无需关心响应中的具体取值。

</Accordion>

<Accordion title="Q2：能不能直接传别的模型名（如本章未列出的）？">

不能。只有 `/v1/models` 端点返回的、且属于音视频分组的模型可用。其他模型名会返回 `model_not_found`。

</Accordion>

<Accordion title="Q3：可以批量出图吗？">

图像 `n` 参数 **仅支持 `1`**。需要批量请在客户端循环发起，便于做并发控制与错误重试。

</Accordion>

<Accordion title="Q4：参考图 / 图生视频怎么传？">

- 图像编辑：走 `POST /v1/images/edits`（multipart/form-data 上传图片二进制）；
- 视频参考图（`veo-3.1-ref` / `kling-3.0`）：将图片作为内容块嵌入 `messages` 数组，具体格式参考控制台「模型广场」中各模型的示例代码。

</Accordion>

<Accordion title="Q5：`quality=auto` 或其他未列出的值会怎样？">

未识别的 `quality` 值将走默认档 `medium`。请使用文档列出的标准取值，避免歧义。

</Accordion>

<Accordion title="Q6：`size` 和 `aspect_ratio` 同时传？">

`size` 派生出的 `aspect_ratio` 会**覆盖**显式传入的 `aspect_ratio`，建议只选用一种方式：

- 完全遵循 OpenAI 协议 → 仅传 `size`；
- 需要 4K、超长视频、非协议比例 → 使用 `quality` / `aspect_ratio` / `duration` 扩展字段。

</Accordion>

<Accordion title="Q7：`kling-3.0` 一直报 reference_images required？">

`kling-3.0` 是图生视频模型，**必须**附带至少一张参考图。如需纯文生视频，请改用 `kling-o3` / `sora-2` / `veo-3.1`。

</Accordion>

</Accordions>
