## 先看这张表

| 你要做什么 | 推荐接口 | 令牌分组 |
| --- | --- | --- |
| 文生图 | `POST /v1/images/generations` | `音视频模型` |
| 图生图 / 图片编辑 | `POST /v1/images/edits` | `音视频模型` |
| 视频生成 | `POST /v1/videos` | `音视频模型` |
| 查询视频进度 | `GET /v1/videos/{video_id}` | `音视频模型` |
| 下载视频文件 | `GET /v1/videos/{video_id}/content` | `音视频模型` |

<Callout type="warn" title="新手最常见错误">
图片和视频模型必须使用 `音视频模型` 分组令牌。不要用 Codex、Claude、Gemini 分组令牌调用这些模型。
</Callout>

## 图片调用

<Tabs items={['cURL', 'Python SDK', 'Node.js SDK']}>
<Tab value="cURL">

```bash
curl https://newapi.prorisehub.com/v1/images/generations \
  -H "Authorization: Bearer sk-音视频分组令牌" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "nano-banana",
    "prompt": "一只柴犬在草地上奔跑，油画风格",
    "n": 1,
    "aspect_ratio": "16:9",
    "quality": "medium"
  }'
```

</Tab>
<Tab value="Python SDK">

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-音视频分组令牌",
    base_url="https://newapi.prorisehub.com/v1",
    timeout=240,
)

resp = client.images.generate(
    model="nano-banana-pro",
    prompt="一只柴犬在富士山下奔跑，浮世绘风格",
    n=1,
    extra_body={
        "aspect_ratio": "16:9",
        "quality": "high",
    },
)

print(resp.data[0].url)
```

</Tab>
<Tab value="Node.js SDK">

```js
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "sk-音视频分组令牌",
  baseURL: "https://newapi.prorisehub.com/v1",
  timeout: 240000,
});

const resp = await client.images.generate({
  model: "gpt-image-2",
  prompt: "一张未来城市夜景海报",
  n: 1,
  aspect_ratio: "16:9",
});

console.log(resp.data[0].url);
```

## 视频调用

视频建议走异步任务：**创建任务 → 查询进度 → 下载 mp4**。

<Tabs items={['1. 创建任务', '2. 查询进度', '3. 下载视频']}>
<Tab value="1. 创建任务">

```bash
curl https://newapi.prorisehub.com/v1/videos \
  -H "Authorization: Bearer sk-音视频分组令牌" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance2.0-5s",
    "prompt": "一只海豚在落日海面跃出水面，电影感",
    "seconds": 5,
    "aspect_ratio": "16:9"
  }'
```

响应示例：

```json
{
  "id": "video_xxx",
  "status": "queued"
}
```

</Tab>
<Tab value="2. 查询进度">

```bash
curl https://newapi.prorisehub.com/v1/videos/video_xxx \
  -H "Authorization: Bearer sk-音视频分组令牌"
```

响应示例：

```json
{
  "id": "video_xxx",
  "status": "completed",
  "progress": 100
}
```

常见状态：

| 状态 | 含义 |
| --- | --- |
| `queued` | 已排队 |
| `processing` | 正在生成 |
| `completed` | 已完成 |
| `failed` | 失败 |

</Tab>
<Tab value="3. 下载视频">

```bash
curl -L https://newapi.prorisehub.com/v1/videos/video_xxx/content \
  -H "Authorization: Bearer sk-音视频分组令牌" \
  -o output.mp4
```

完成后内容接口返回 `video/mp4`，可以直接给浏览器、Pro Lab 或其他前端播放器使用。

</Tab>
</Tabs>

<Callout type="warn" title="视频生成很慢是正常的">
视频通常需要几十秒到几分钟。客户端超时时间建议设置为 5 分钟以上，不要按聊天接口的 30 秒超时来处理。
</Callout>

## 常用参数

| 参数 | 适用 | 示例 | 说明 |
| --- | --- | --- | --- |
| `model` | 全部 | `seedance2.0-5s` | 模型名，必须在 `音视频模型` 分组里 |
| `prompt` | 图片 / 视频 | `一只海豚跃出水面` | 生成描述 |
| `n` | 图片 | `1` | 目前只建议传 `1` |
| `aspect_ratio` | 图片 / 视频 | `16:9`、`9:16`、`1:1` | 画面比例 |
| `quality` | 部分图片 / Veo | `low`、`medium`、`high` | 清晰度档位 |
| `size` | 图片 | `1024x1024` | OpenAI 标准尺寸字段 |
| `seconds` | 固定时长视频 | `5`、`10` | Seedance / Kling 固定秒数模型建议使用 |
| `duration` | 通用视频 | `4`、`8`、`12` | Sora / Veo / Kling 3.0 等模型建议使用 |
| `image` / `image_url` | 图生图 / 图生视频 | 图片 URL | 需要参考图时使用，具体字段以客户端支持为准 |

## 参数建议

| 模型 | 比例建议 | 时长建议 |
| --- | --- | --- |
| `seedance2.0-5s` | `16:9` / `9:16` | `seconds: 5` |
| `seedance2.0-10s` | `16:9` / `9:16` | `seconds: 10` |
| `kling-5s` | `16:9` / `9:16` | `seconds: 5`，建议带首帧图 |
| `kling-10s` | `16:9` / `9:16` | `seconds: 10`，建议带首帧图 |
| `sora-2` / `sora-2-pro` | `16:9` / `9:16` | `duration: 4/8/12` |
| `veo-3.1*` | `16:9` / `9:16` | `duration: 4/6/8` |

<Callout type="info" title="自动化调用建议">
如果你在 Pro Lab 或自己的程序里自动调用视频，请优先使用 `/v1/videos`。这个接口的状态、进度和下载地址都更清楚，适合前端轮询和失败重试。
</Callout>
