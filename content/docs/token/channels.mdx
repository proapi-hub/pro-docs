## 这页是做什么的

“渠道”可以理解为 ProAPI 后面真正连接模型服务的线路。你平时只需要创建令牌、选择分组、填写模型名；但当你想判断“为什么这个模型能不能用”“哪个分组更稳”时，就需要看渠道。

本页来自当前线上后台快照，更新时间：**2026-05-27**。

| 渠道总数 | 24 条 |
| 启用渠道 | 18 条 |
| 禁用渠道 | 6 条 |
| 模型价格表 | 196 个模型 |
| 当前分组 | `default`、`Codex-Pro`、`Codex-Plus-Pro 混池`、`claude-max-满血`、`Ant-Claude`、`Kiro-Claude-高缓存`、`claude-reverse-高缓存`、`Windsurf`、`Ant-Gemini`、`Grok`、`音视频模型` |

<Callout type="warn" title="普通用户重点看分组，不需要背渠道名">
渠道名主要用于管理员排错。普通用户只需要确认：令牌选了正确分组，模型广场里能看到目标模型。
</Callout>

## 排错时怎么用这页

| 现象 | 优先检查 |
| --- | --- |
| 提示模型不存在 | 令牌分组是否包含该模型；模型广场是否还能看到它 |
| Claude Code 不稳定 | 优先换到 `claude-max-满血` 或 `Kiro-Claude-高缓存` |
| Gemini CLI 失败 | 确认令牌分组是 `Ant-Gemini`，不要用 `音视频模型` 令牌 |
| 图片/视频模型失败 | 确认令牌分组是 `音视频模型`，并查看 [音视频生成 API](/api/media) 的模型名和参数 |
| Grok 费用或可用性异常 | 以 `Grok` 分组和模型广场实时价格为准，旧的 `x0.005` 说明已过期 |
