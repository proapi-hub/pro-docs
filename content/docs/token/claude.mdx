## 怎么选 Claude 分组

当前真实可用的 Claude 相关分组有 **3 个**(2026-06-06 快照,以模型广场为准):

| 分组 | 当前倍率 | 稳定性 | 适合场景 |
| --- | --- | --- | --- |
| `claude-max-满血` | `x1.6` | ⭐⭐⭐⭐⭐ | 官方满血 20x |
| `claude-awsq-满血` | `x0.6` | ⭐⭐⭐⭐⭐ | 官方 aws-awsq 渠道,95% 高缓存,重要任务+性价比兼顾 |
| `Kiro-Claude-高缓存` | `x0.2` | ⭐⭐⭐⭐ | Claude Code 日常主力,高缓存性价比首选 |

<Callout type="warn" title="已移除分组">
旧文档中的以下分组当前已不在真实分组列表中,对应上游渠道也已禁用,请不要再创建或配置:

- `Ant-Claude`(原 x0.4) 下架原因【反重力claude 质量低速率慢】
- `claude-reverse-高缓存`(原 x0.1)下架原因【经常容易调用失败,不稳定的高缓存渠道】

- `claude-max-特价-满血`(原0.6)下架原因【aws**** 官方风控严重，成本偏高】****

如果你的令牌仍指向以上分组,请前往控制台编辑令牌切换到现行分组。
</Callout>

### claude-max-满血

- **倍率:** `x1.6`
- **定位:** Claude 官方满血 20x 高稳定渠道。
- **适合:** Claude Code 中的长任务、生产环境、重要代码修改、复杂 Agent 工作流。
- **当前主要模型(2026-06-06 快照):**
  - `claude-opus-4-7`
  - `claude-opus-4-6`
  - `claude-opus-4-5-20251101`
  - `claude-opus-4-1-20250805`
  - `claude-opus-4-20250514`
  - `claude-sonnet-4-6`
  - `claude-sonnet-4-5-20250929`
  - `claude-sonnet-4-20250514`
  - `claude-3-7-sonnet-20250219`
  - `claude-3-5-sonnet-20241022`
  - `claude-3-5-sonnet-20240620`
  - `claude-haiku-4-5-20251001`
  - `claude-3-5-haiku-20241022`

<Callout type="warn" title="只能用于 Claude Code">
该分组对应官方 20x 满血账号资源,**仅供 Claude Code 工具调用**,不支持外接到其他客户端、Agent 框架或代码集成。需要外接请使用 `claude-awsq-满血` 或 `Kiro-Claude-高缓存`。
</Callout>

</Tab>
<Tab value="claude-awsq-满血">

### claude-awsq-满血

- **倍率:** `x0.6`
- **定位:** Claude 官方 aws-awsq 渠道,**95% 高缓存**,稳定档与性价比兼顾。
- **适合:** 重要 Claude 任务但又想控制成本;支持外接到 Claude Code、Cline、Cursor 等所有工具。
- **当前主要模型(2026-06-06 快照):**
  - `claude-opus-4-8`
  - `claude-opus-4-7`
  - `claude-opus-4-6`
  - `claude-opus-4-5-20251101`
  - `claude-sonnet-4-6`
  - `claude-haiku-4-5-20251001`

<Callout type="info" title="新手怎么理解">
和 `claude-max-满血` 同为最稳定档(⭐⭐⭐⭐⭐),价格只有它的约 1/3,而且 95% 缓存命中率显著降低长对话成本。**如果你需要外接到 Claude Code 以外的客户端,这是首选。**
</Callout>

</Tab>

### Kiro-Claude-高缓存

- **倍率:** `x0.2`
- **定位:** Claude 高缓存线路,当前站内 Claude 性价比最高的一档。
- **适合:** Claude Code 日常开发、文档理解、翻译、规划任务。
- **当前主要模型(2026-06-06 快照):**
  - `claude-opus-4-8`
  - `claude-opus-4-7`
  - `claude-opus-4-6`
  - `claude-opus-4-5-20251101`
  - `claude-sonnet-4-6`
  - `claude-sonnet-4-5-20250929`
  - `claude-haiku-4-5-20251001`

<Callout type="info" title="推荐顺序">
日常 Claude Code 任务先用 `Kiro-Claude-高缓存`;遇到稳定性要求很高或缓存命中率不高的长任务,再切 `claude-awsq-满血`;只在 Claude Code 工具内使用最高档时切 `claude-max-满血`。
</Callout>

## Claude Code 配置提醒

- Base URL 请按对应教程填写,不要把普通 OpenAI `/v1` 地址误填到 Anthropic 原生配置里。
- 如果工具支持 Anthropic 原生协议,优先使用 `/v1/messages`。
- 如果只是 OpenAI 兼容客户端,也可以走 `/v1/chat/completions`,但工具调用和缓存能力可能不如原生协议完整。

</Callout>
