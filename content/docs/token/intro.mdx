## 先记住一句话

**令牌分组 = 你这把 API Key 可以使用哪些模型 + 按什么倍率扣额度。**

创建令牌时一定要选对分组。分组选错时，最常见的表现是：

- 调用时报 `model not found` 或“模型不存在”。
- 明明模型广场能看到模型，但你的令牌无法调用。
- 可以调用，但倍率不是你预期的那一档。

<Callout type="info" title="给新手的建议">
如果你只是想最快跑通，先用对应用途的专用分组：写代码选 `Codex-Plus-Pro 混池`，Claude Code 选 `Kiro-Claude-高缓存` 或 `claude-max-满血`，Gemini CLI 选 `Ant-Gemini`，画图/视频选 `音视频模型`。
</Callout>

## 怎么看“倍率”

倍率只决定在模型标准价格基础上再乘多少。

举例：

| 模型标准消耗 | 分组倍率 | 实际扣除 |
| --- | --- | --- |
| 10 额度 | `x1` | 10 额度 |
| 10 额度 | `x0.5` | 5 额度 |
| 10 额度 | `x0.1` | 1 额度 |
| 10 额度 | `x1.2` | 12 额度 |

<Callout type="info" title="不要只看倍率">
贵模型即使用低倍率，单次消耗也可能高；便宜模型即使用 `x1`，也可能很省。最终请以模型广场显示的模型价格为准。
</Callout>

## 如何查看最新模型

站点里的模型和渠道会随上游情况调整，**控制台的“模型广场”永远优先于文档快照**。

1. 登录控制台。
2. 点击右上角 **模型广场**。
3. 左侧选择令牌分组。
4. 右侧查看该分组下可调用的模型、价格和可用状态。

<Callout type="warn" title="为什么要先看模型广场">
很多调用失败不是代码问题，而是令牌分组没有这个模型。调用前先看模型广场，可以少走很多弯路。
</Callout>

## 目前有哪些真实渠道

当前后台共有 **24 条渠道**：

- **18 条启用**：正常参与路由。
- **6 条禁用**：保留配置，但不会参与正常调用。
- 分组覆盖：Codex、Claude、Gemini、Grok、音视频模型、Windsurf 与 default。

想看每条渠道对应哪个分组、有哪些模型，请阅读 [当前渠道总览](/token/channels)。
